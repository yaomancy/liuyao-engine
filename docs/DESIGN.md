# 设计取舍 / Design rationale

> 这篇讲的不是「引擎怎么用」，而是「**为什么这么设计**」——背后的领域判断与工程决策。
> 代码能证明「算得对」（见 README「为什么可信」），但**为什么是这套结构**，要在这里说清楚。

## 一条主线：凡进核心，必可被独立交叉验证

整个引擎只产「**硬事实**」（排盘 / 生克 / 旺衰），不做解读。在此之上还有一条更硬的线：

> **只有能被独立实现交叉验证的东西，才允许进核心层。**

凡是「有争议、无统一真值、无法差分验证」的内容——神煞、解读、流派附加——要么分层隔离、显式开启，要么交给外壳 / AI，**绝不混入核心的硬事实**。下面三个决定，都是这条线的推论。

---

## ① 为什么神煞不进核心

**问题**：神煞（桃花、驿马、羊刃、天乙贵人……）流派各异、取用规则互相矛盾，**没有统一真值**，也**没有独立的开源实现可供差分比对**。

**取舍**：核心层只放能被 najia 等独立实现交叉验证的硬事实（纳甲 / 六亲 / 六神 / 世应 / 旬空 / 旺衰）。神煞无法被差分验证 → **不进核心**。

**立场**：这不是否定神煞。本引擎断法遵《增删卜易》，而该书本就以五行生克制化、旺衰、用神为主，对神煞多持保留。若要使用神煞，应放在**可选 / 展示层**，明确标注其流派来源，绝不与核心「硬事实」混淆——否则会把不可证伪的内容污染进确定性结果。

---

## ② 为什么用 najia 做差分预言机（oracle）

**问题**：命理类代码最大的陷阱是「**自验证悖论**」——自己写的引擎，拿什么证明算得对？人工专家终审不可规模化、不可复现、本身也会错。

**取舍**：引入一个**独立、开源、可程序化运行**的第二实现作为「预言机」，对海量「(卦象 + 时间)」组合做自动差分比对，不一致即视为 bug，并随 CI 反复复现。

- 选 [`najia`](https://github.com/bopo/najia)（MIT）作**主预言机**：① 许可证可商用合规；② 程序化排盘，可被脚本驱动跑大规模组合；③ 社区已验证。
- 再叠 [`yigram-najia-rules`](https://github.com/AdrienSterling/yigram-najia-rules)（MIT，机器可读规则表）作**第二独立来源**：纳甲 / 六亲 / 六神 / 世应 / 旬空 / 宫五行 / 地支五行**三方一致**才采信，降低「两个实现共享同一错误」的风险。

**价值**：把「正确性」从「信某位专家」变成「CI 里任何人都能反复复现的工程事实」。锚点卦例的标准答案也以两书原例 + 多工具差分为准，不依赖人工终审。

---

## ③ 为什么历法要双库交叉（sxtwl + lunar-python）

**问题**：六爻一切从「干支四柱」起算，**历法是零容错的地基**——日柱错一天，旺衰 / 旬空 / 月建全错，整盘皆废。

**坑都在边界**：

- **月建按节气分界**（非农历初一）：交节的**精确时刻**决定属上月还是下月。
- **子时换日**：23:00 起算次日日柱（子初换日，术数主流惯例），非 00:00。
- **跨时区起卦**：海外求测人当地墙钟与节气所在的绝对时刻须分别处理。

**取舍**：以 [`sxtwl`](https://github.com/yuangu/sxtwl_cpp)（寿星天文历，高精度天文算法）做主算，[`lunar-python`](https://github.com/6tail/lunar-python)（另一套独立实现）做交叉验证；交节 / 子时 / 跨时区等边界 case 都有单测交叉比对，不一致即暴露。

**当前的时间处理**（见 [`liuyao/calendar.py`](../liuyao/calendar.py)）：

- **日柱 / 时柱**取求测人**当地墙钟**（「当时当地」），子时换日后用换日后日干起五鼠遁，时支恒为子。
- **年 / 月柱**的交节是**全球绝对事件**：先把本地时刻整体换算为等效北京民用时刻（含日期），再按交节精确时刻判定，使任意时区下的同一绝对瞬间得到相同年 / 月柱。

### 关于真太阳时（true solar time）

六爻起卦取「掷钱此刻」。本引擎默认用**按时区的常用钟表时间**，**不默认**做经度 + 均时差校正——这是有意为之的取舍，原因有二：

1. **法度**：六爻重「起卦之机」，《增删卜易》一路传统多用所在地常用时，与八字 / 子平强调真太阳时校正经度的立场不同。
2. **可验证性**：强制真太阳时会引入「取哪条经度、用哪套均时差公式」这类**无统一真值、无法差分验证**的不确定，违背本引擎「凡进核心必可交叉验证」的主线。

真太阳时并非不能用，而是将作为**可选层**提供：后续会开放可选的经度 + 均时差校正输入（与神煞同理——分层隔离、显式开启），由用户 / 外壳决定是否启用，**不改变核心的可验证性**。

---

## ④ 「不混派」也是一个正确性决定

混派 = 把互相矛盾的流派规则塞进同一引擎，结果**不可证伪**。本引擎**装卦遵《卜筮正宗》、断法遵《增删卜易》**，守一套法度，才能有唯一真值、才能被差分验证。「不混派」在这里不是教条，而是「正确性可证明」的前提。

---

## 收束

以上取舍有同一个共同点：

> **凡进核心的，必可被独立交叉验证；凡不可验证或有争议的（神煞、真太阳时、解读），要么分层隔离、显式开启，要么交给外壳 / AI——绝不混入硬事实。**

这就是这个引擎敢叫「确定性引擎」的底气。
